# 001. Docker для воспроизводимой среды

**Status:** Accepted

## Context

Команда из трёх человек работает на разных ОС (Mac, Windows, Linux). Проект использует Python-библиотеки с C-расширениями (`cryptography`, `hashpumpy`), которые компилируются по-разному на разных платформах. Есть риск ситуации «у меня работает» с разными версиями Python, OpenSSL и системных библиотек.

Дополнительно: `hashpumpy` требует системный `libssl-dev` при сборке. На Windows это нетривиально.

## Decision

Запускать все демки и тесты внутри Docker-контейнера на базе `python:3.11-slim` с Ubuntu под капотом. Все команды через `make demo-XX` автоматически используют контейнер.

Разработка: можно писать код в любом редакторе, изменения видны через volume mount.

## Consequences

**Лучше:**
- Одна среда для всех трёх участников
- CI использует тот же образ — нет расхождения «локально работает, CI ломается»
- Onboarding нового участника: `make setup` — единственная команда

**Хуже:**
- Нужен Docker Desktop на Windows (бесплатен только для некоммерческого использования)
- Первый `docker build` медленный (~2–5 мин на slow интернете)
- Изменения в `requirements.txt` требуют пересборки образа

**Trade-off принят:** надёжность важнее скорости setup.
