# Demo 06 — MD5 Collision

**Владелец:** Kirill | **Блок:** Хэши, MACs, подписи

## What it demonstrates

Два разных 128-байтовых блока данных, дающих одинаковый MD5-хэш `79054025255fb1a26e4bc422aef54eb4`. Это коллизия Wang et al. (2004) — первая практическая атака на MD5, завершившаяся за несколько часов на PC. Блоки отличаются в 3 байтах (изменение 3 бит достаточно для коллизии через дифференциальный криптоанализ).

## The math

MD5 — итеративная конструкция Merkle-Damgård. Внутреннее состояние (128 бит) обновляется блоками по 512 бит через 4 раунда нелинейных функций. Wang et al. нашли **дифференциальную пару**: два сообщения `m, m'` где `m ⊕ m' = Δ` (специально подобранная разница), такие что после обработки обоих блоков внутренние состояния совпадают. Вычислительная сложность: ~2²⁴ операций (часы на современном PC).

**Chosen-prefix коллизия** (Stevens et al., 2009) — более мощная версия: для любых двух префиксов `P1, P2` найти суффиксы `S1, S2` такие что `MD5(P1||S1) == MD5(P2||S2)`. Использована в атаке **Flame** (2012): два разных публичных ключа, один подписанный сертификат от Microsoft.

## How to fix

- **Нельзя использовать MD5** для цифровых подписей, сертификатов, MAC, идентификации файлов в security-контексте
- **Минимум:** SHA-256 (устойчив к коллизиям на уровне 2¹²⁸)
- **Лучше:** SHA-3 (Keccak, губчатая конструкция — другой принцип, нет Merkle-Damgård слабости)
- **Для скорости + безопасность:** BLAKE3 (параллельный, ~1 GB/s, без известных атак)
- MD5 допустимо только для checksums без security-требований (например, дедупликация файлов в закрытой системе)
