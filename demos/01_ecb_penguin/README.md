# Demo 01 — ECB Penguin

**Владелец:** Osman | **Блок:** Симметричное шифрование

## What it demonstrates

AES в режиме ECB (Electronic Codebook) шифрует каждый 16-байтовый блок независимо с одним и тем же ключом. Это означает: **одинаковые блоки открытого текста → одинаковые блоки шифртекста**. Когда изображение состоит из больших однородных областей (как логотип Tux), структура «просвечивает» сквозь шифрование. CBC и GCM визуально неотличимы от случайного шума.

## How to run

```bash
# Подготовка: скачать Tux PNG и положить в assets/
# https://commons.wikimedia.org/wiki/File:Tux.svg → конвертировать в PNG
cp /path/to/tux.png demos/01_ecb_penguin/assets/tux.png

# Запуск (если нет Tux.png, создастся синтетическая замена)
make demo-01
# или напрямую:
python demos/01_ecb_penguin/demo.py
```

## Expected output

```
[*] Загружено изображение: 128x128 px, 49152 байт
[+] Сохранено: tux_ecb.png, tux_cbc.png, tux_gcm.png

  ECB → структура видна: одинаковые пиксели = одинаковые блоки шифртекста
  CBC → случайный шум: IV+цепочка XOR убирает паттерны
  GCM → случайный шум: counter-mode keystream, плюс аутентификация
```

В `assets/` появятся три файла. Откройте их в любом просмотрщике — в `tux_ecb.png` силуэт будет виден.

## The math

**ECB:** `C_i = AES_K(P_i)` — каждый блок независимо.

**CBC:** `C_i = AES_K(P_i ⊕ C_{i-1})`, где `C_0 = IV` — каждый блок XOR-ится с предыдущим шифртекстом, что убивает паттерны.

**GCM (AEAD):** CTR-режим + Poly1305 аутентификация. `C_i = P_i ⊕ AES_K(nonce || counter_i)` — keystream генерируется шифрованием счётчика, шифртекст неотличим от случайного.

Почему ECB провалился: если `P_1 == P_2`, то `C_1 == C_2`. В изображении соседние пиксели похожи → блоки открытого текста повторяются → блоки шифртекста повторяются.

## How to fix

Никогда не используйте ECB для данных с паттернами (изображения, JSON, структурированные форматы). Используйте **AES-GCM** или **ChaCha20-Poly1305** (оба — AEAD, дают аутентификацию в одном примитиве). CBC допустимо только в связке с отдельным HMAC (Encrypt-then-MAC).
